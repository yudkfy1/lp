# LP作成プロジェクト運用ルール

このディレクトリは、複数クライアントのLP（ランディングページ）を制作・GitHub → Vercelでデプロイするための作業場所です。

## リポジトリ構成
- 1 GitHubリポジトリ（`yudkfy1/lp`） × 1 Vercelプロジェクト で運用
- LPごとに **サブフォルダ** を切って追加する（過去のLPは絶対に消さない・上書きしない）
- URL構成例: `lp-xxxx.vercel.app/クライアント名/`

```
lp/
├── index.html          ← ポートフォリオ入口（各LPへのリンク）
├── koen/               ← 麻辣湯LP
├── minori/             ← 八百屋LP
└── クライアント名/      ← 新規LPはココに追加
```

## 【最重要】LP制作依頼を受けたら、必ず最初にこの質問をする

ユーザーから新規LP作成の依頼を受けたとき、コードを書き始める前に **必ず** 以下を聞くこと：

> **「今回は "本格的な制作" ですか？それとも "イメージ確認/テスト用" ですか？」**

回答に応じて、以下のモードに分岐する。

---

### モードA: 本格制作モード（本番案件）

**方針**: ヒアリングレベルMAX。以下の全項目を段階的に、しかし**びしばし**質問する。曖昧な回答は必ず具体化を要求する。憶測でLP制作を始めない。

#### ① 基本情報
- クライアント名 / 店舗名 / ブランド名
- 業種・商材の概要
- なぜLPを作るのか（背景・課題）
- 納期・予算感

#### ② ターゲット・ペルソナ（最重要）
- ターゲット層（年齢・性別・職業・年収・居住地・家族構成）
- ペルソナの具体像（1人の人物として言語化する）
- ペルソナの悩み・欲求・恐れ
- 購入/申込を決断するシチュエーション
- ペルソナがLPに辿り着くまでの導線（広告？SNS？検索？）

#### ③ 目的・ゴール（KGI/KPI）
- LPの主目的（購入 / 問い合わせ / 資料DL / LINE登録 / 来店予約 / メルマガ登録）
- 目標CV数（月間・週間）
- CVボタンの飛び先（フォーム？LINE？電話？外部予約サイト？）
- 計測手段（GA4 / GTM / Meta Pixel / 計測不要）

#### ④ 訴求内容（提供する情報）
- 商品/サービスの特徴・USP（他にない強み）
- 価格・プラン構成
- 競合サービスとの差別化点
- 実績・受賞歴・メディア掲載
- お客様の声・レビュー（実在／架空か）
- 保証・返金ポリシー
- 想定される反論・不安（それに対する回答）

#### ⑤ デザイン方針
- ブランドカラー（メイン・アクセント）
- ロゴの有無（提供可能か）
- 参考にしたいLP・サイト（3つ程度、URLで）
- デザインの雰囲気（高級感/親しみ/クール/かわいい/シック/ポップ）
- NG表現・NGカラー
- フォントの希望（明朝/ゴシック/手書き）

#### ⑥ 素材
- 写真素材の提供可否（提供 / フリー素材OK / 撮影必要）
- ロゴファイル（ai / svg / png）
- 動画素材の有無
- キャッチコピー案の有無

#### ⑦ 必須セクション
- 会社概要 / 特定商取引法 / プライバシーポリシー / FAQ / お問い合わせフォーム / アクセス / SNSリンク
- 対応言語（日本語のみ / 多言語）

#### ⑧ 技術要件
- お問い合わせフォームの送信先（メール直送 / Googleフォーム / Formspree / 独自バックエンド）
- 独自ドメインの有無・希望
- SEOキーワード（検索されたい語句）
- OGP画像・meta情報
- モバイル対応レベル

#### ⑨ 運用・保守
- 公開後の更新頻度
- 更新担当（クライアント自身 / 制作側継続）
- 保守契約の有無

---

**ヒアリングの進め方**:
- 一度に全部聞かず、**AskUserQuestion** で3〜4項目ずつまとめて聞く
- 「わからない」「おまかせ」と言われた項目は、提案+その提案の理由を明示して確認を取る
- ヒアリング完了後、**制作方針を1〜2文でまとめて最終確認** → OKが出てから着手

---

### モードB: イメージ/テストモード

**方針**: ヒアリング最小限。以下のみ確認したら即着手。

- 題材（何のLPか、業種・商材）
- 雰囲気の方向性（1〜2語で / 例：高級・ポップ・辛い・和風）
- 特に重視したい訴求点（あれば）

その他はこちらの裁量で完成させる。

---

## 制作フロー（両モード共通）

1. ヒアリング完了 → 制作方針を1文で最終確認
2. `/クライアント名/` フォルダを新規作成
3. index.html + style.css を制作
4. ルートの `index.html`（ポートフォリオ入口）にリンクを追加
5. `git add . && git commit -m "..." && git push`
6. Vercelが自動デプロイ（30秒〜1分）
7. デプロイURLをユーザーに報告

## 認証情報
- GitHubトークンはmacOSキーチェーンに保存済み（`git push` は認証プロンプトなしで通る）
- コミット時のuser.name/emailはグローバル設定済み

## 絶対にやってはいけないこと
- 既存のLPフォルダを上書き・削除しない
- 本格制作モードで、ヒアリング完了前にコードを書き始めない
- 「おまかせで」の一言で本格制作を始めない（必ず主要項目は確認する）
